GitHub Setup Guide üîß

Follow these detailed steps to set up your GitHub repository and connect it to Our Space.

üì¶ Step 1: Create GitHub Repository

  1. Go to github.com and log in to your account
  2. Click the "+" icon in the top-right corner
  3. Select "New repository"
  4. Enter a repository name (e.g., our-space-data)
  5. ‚ö†Ô∏è IMPORTANT: Select "Private" to keep your data secure
  6. Leave "Initialize with README" unchecked
  7. Click "Create repository"
  8. Note your repository name format: your-username/our-space-data

üîë Step 2: Generate Personal Access Token (PAT)

  1. Go to github.com/settings/tokens
  2. Click "Generate new token" dropdown
  3. Select "Generate new token (classic)"
  4. Enter a note/name: Our Space App
  5. Set expiration: Choose "No expiration" or your preferred duration
  6. ‚ö†Ô∏è CRITICAL: Under "Select scopes", check "repo" (this will select all sub-options)
  7. Scroll down and click "Generate token"
  8. üî¥ IMPORTANT: Copy the token immediately (format: ghp_xxxxxxxxxxxxx)
  9. Save it somewhere safe - you won't be able to see it again!

üíæ What Gets Stored in Your Repo?

  • master_user_{username}.txt - Your room list and quick access data
  • {ROOMCODE}_master.txt - Index file for each room
  • {ROOMCODE}_1.txt, _2.txt... - Room data files (split automatically at 50MB)
  • All files are JSON format and human-readable
  • Images stored as base64 inside the files

üîê Security Notes

  • Never share your PAT with anyone
  • PAT stored only in your browser's localStorage
  • Use a strong, unique GitHub password
  • Enable 2FA on your GitHub account for extra security

Login & Account Creation üîÑ

Understanding how the login system works and what happens behind the scenes.

üöÄ First Time Setup (New User)

Step 1: GitHub Connection
  1. Open the app - you'll see "Connect Your GitHub" screen
  2. Enter your repository: username/repo-name
  3. Paste your Personal Access Token
  4. Click "Connect & Continue"
  5. System verifies connection (checks if repo is accessible)
  6. Success! Moves to username screen
Step 2: Choose Username
  1. Enter your desired username (3-30 characters)
  2. Username rules: lowercase, letters, numbers, periods, underscores only
  3. Click "Continue"
  4. System checks if master_user_{username}.txt exists in your repo
  5. If not found ‚Üí Creates new account automatically
  6. Creates master file with your username and empty rooms array
  7. Saves credentials to browser localStorage as github_{username}
  8. Shows room dashboard - ready to create/join rooms!

üîÑ Returning User (Existing Account)

When You Come Back:
  1. Enter same repository and PAT (or auto-loaded if saved)
  2. System verifies GitHub connection
  3. Enter your username
  4. System finds your master_user_{username}.txt file
  5. Loads all your rooms and data
  6. Shows room dashboard with all your rooms
  7. Click any room to enter and start using

üíª Using Multiple Devices

To access your rooms on another device:

  1. Open the app on new device
  2. Enter SAME GitHub repo and PAT
  3. Enter SAME username
  4. All rooms sync automatically
  5. Any changes made sync across all devices every 1.5 seconds

üë• Sharing with Partner/Friend

Option 1: Shared Repository
  • Both use the SAME GitHub repo and PAT
  • Both enter DIFFERENT usernames
  • Creates separate master files for each user
  • When one creates a room, other can join using room code
  • ‚úÖ Best for instant synchronization
Option 2: Separate Repositories
  • Each person uses their OWN GitHub repo and PAT
  • Room creator's repo stores the room data
  • Other person needs access to creator's repo to join
  • ‚úÖ More privacy, but requires repo collaboration setup

Creating & Joining Rooms üö™

Step-by-step guide for creating new rooms and joining existing ones.

‚ûï Creating a New Room

From Room Dashboard:
  1. After login, you'll see "Your Rooms" dashboard
  2. Click "+ Create New Room" button
  3. Enter a Room Name (e.g., "Our Love Story", "Best Friends Forever")
  4. Choose Room Type:
    • Click üíï Love Room for romantic relationships
    • Click üë´ Friendship for platonic friendships
  5. Click "Create Room"
  6. System generates unique 6-character room code
  7. Creates room files in your GitHub repository:
    • {ROOMCODE}_master.txt - Room index
    • {ROOMCODE}_1.txt - Room data
  8. Adds room to your master file
  9. You become the room admin automatically
  10. Room opens - ready to add content!

üì® Inviting Others (Sharing Room Code)

How to Share:
  1. After creating room, note the 6-character code (shown in success message)
  2. Or go to Settings page to view room code
  3. Share this code with your partner/friend via:
    • Text message
    • Email
    • In person
    • Any secure communication method
  4. ‚ö†Ô∏è Keep room code private - anyone with code can request to join

üîó Joining an Existing Room

Join Process:
  1. Get room code from friend/partner (6 characters)
  2. From room dashboard, click "Join Room"
  3. Enter the room code (automatically converts to uppercase)
  4. Click "Request to Join"
  5. System checks if room exists in connected GitHub repo
  6. If found, sends join request to room admin
  7. You'll see "Join request sent!" message
  8. Now wait for admin approval...

üëë Admin: Approving Join Requests

Approval Process:
  1. When someone requests to join, you'll see:
    • üîî Bell icon appears in top-right corner
    • Red badge shows number of pending requests
  2. Click the üîî bell icon
  3. Modal opens showing all join requests
  4. Each request shows:
    • Username of requester
    • Date/time of request
    • Two buttons: Approve and Deny
  5. To Approve:
    • Click "Approve" button
    • User added to room members list
    • System message posted in chat: "{username} joined the room"
    • User's master file updated with room info
    • They can now access and use the room
  6. To Deny:
    • Click "Deny" button
    • Request removed from queue
    • User receives no notification (silent denial)

‚ö†Ô∏è Important Notes

  • Repository Access: For joining to work:
    • Both users must use SAME GitHub repository, OR
    • Joiner must have access to creator's repository
  • No Limit: Create and join unlimited rooms
  • One Admin: Room creator is admin (transfers if admin leaves)
  • Room Codes: Unique, case-sensitive, 6 characters

Advanced Tips & Best Practices üí°

Pro tips to get the most out of Our Space and avoid common issues.

üîê Security Best Practices

‚ö†Ô∏è Critical Security Tips:
  • Never Share Your PAT:
    • Personal Access Token = full access to your GitHub
    • Don't share it via text, email, or screenshots
    • Each person should have their own PAT
  • Keep Repository Private:
    • Always use private repos for your data
    • Public repos = anyone can see your memories
    • Check repo visibility in GitHub settings
  • Use Strong GitHub Password:
    • Unique password not used elsewhere
    • Enable 2-factor authentication (2FA)
    • Protects your GitHub account from unauthorized access
  • Room Code Privacy:
    • Only share room codes with intended members
    • Anyone with code can request to join
    • You can deny unwanted requests, but prevention is better
  • Browser Security:
    • Don't use on public/shared computers
    • Clear browser data if using shared device
    • Your credentials stored in localStorage

üíæ Data Management

Understanding File Sizes:
  • Images:
    • Maximum 5MB per image (enforced by app)
    • Stored as base64 (increases size by ~33%)
    • Tip: Compress images before uploading
    • Use tools like TinyPNG or built-in photo editor
  • File Splitting:
    • App auto-splits data at 50MB per file
    • Creates multiple numbered files: _1.txt, _2.txt, etc.
    • Master file tracks all split files
    • Transparent to you - loads automatically
  • GitHub Limits:
    • Free accounts: 1GB storage total
    • Individual file limit: 100MB (we stay well under)
    • Monitor your repo size in GitHub settings
Backup Strategies:
  • Automatic GitHub Backup:
    • GitHub itself is your primary backup
    • All commits tracked in Git history
    • Can view/restore previous versions
  • Additional Backup:
    • Clone your repo locally: git clone [repo-url]
    • All your data downloaded as JSON files
    • Store on external drive or cloud storage
  • Export Memories:
    • Download files directly from GitHub
    • JSON format - can open in any text editor
    • Extract images from base64 if needed
